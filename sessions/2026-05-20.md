## STEP7 — 루틴/알람 CRUD + Spring Scheduler + 알림 이력 DB 기록

### 새로 생성한 파일

| 파일 | 설명 |
|------|------|
| `io.bitpet.routine.domain.RoutineType` | Enum: FEEDING / CLEANING / CUSTOM |
| `io.bitpet.routine.domain.RoutineMst` | 루틴 엔티티 (petId, routineType, title, cycleDays, lastExecutedAt, nextDueAt, `boolean active`, memo) |
| `io.bitpet.routine.domain.AlarmMst` | 알람 엔티티 (routineId, `LocalTime alarmTime`, `boolean enabled`) |
| `io.bitpet.notification.domain.NotificationStatus` | Enum: PENDING / SENT / FAILED / READ |
| `io.bitpet.notification.domain.NotificationLogDtl` | 알림 이력 엔티티 (userId, petId, routineId, title, body, sentAt, status, errorMessage) |
| `io.bitpet.routine.repository.RoutineMstRepository` | `findOverdueRoutines(@Param("now") Instant now)` JPQL 쿼리 |
| `io.bitpet.notification.repository.NotificationLogDtlRepository` | `findTop50ByUserIdOrderBySentAtDesc` |
| `io.bitpet.routine.service.RoutineService` | 루틴 CRUD + 알람 CRUD + executeRoutine |
| `io.bitpet.notification.service.NotificationService` | `@Transactional(REQUIRES_NEW)` 알림 생성 |
| `io.bitpet.scheduler.RoutineScheduler` | `@Scheduled(fixedDelay=60_000)` — 만기 루틴 스캔 → 알림 기록 → next_due_at 전진 |
| `io.bitpet.common.config.SchedulerConfig` | `@EnableScheduling` |
| `io.bitpet.routine.controller.RoutineController` | `/api/v1/pets/{petId}/routines/**`, `/api/v1/routines/{id}/alarms/**` |
| `io.bitpet.notification.controller.NotificationController` | `GET /api/v1/notifications`, `PATCH /api/v1/notifications/{id}/read` |

### 핵심 설계 결정

- **물리 삭제**: RoutineMst, AlarmMst는 V5 스키마에 `deleted_at` 없음 → soft delete 미적용
- **트랜잭션 격리**: NotificationService에 `REQUIRES_NEW`로 루틴별 알림 실패가 전체 스케줄러 트랜잭션을 롤백하지 않도록
- **boolean 필드명**: `active` / `enabled` (isActive 쓰면 Lombok이 `isIsActive()` 생성하는 버그 회피)
- **next_due_at 전진**: 알림 생성 성공·실패 무관하게 항상 전진 (중복 알림 방지)

### 수정한 파일

- `ErrorCode.java` — `ROUTINE_NOT_FOUND`, `ALARM_NOT_FOUND`, `ROUTINE_ACCESS_DENIED`, `NOTIFICATION_NOT_FOUND` 추가


### 새로 생성한 파일

| 파일 | 설명 |
|------|------|
| `localstack-init/01-create-bucket.sh` | LocalStack 기동 시 `bitpet-dev` 버킷 자동 생성 + CORS 설정 |
| `io.bitpet.common.config.S3Properties` | `@ConfigurationProperties("bitpet.s3")` record |
| `io.bitpet.common.config.S3Config` | S3Client + S3Presigner 빈 (pathStyleAccess=true for LocalStack) |
| `io.bitpet.storage.S3Service` | presignPut / presignGet / deleteObject |
| `io.bitpet.photo.domain.PetPhotoDtl` | 사진 엔티티 (petId, s3Key, fileSize, mimeType, width, height, takenAt, caption) |
| `io.bitpet.photo.service.PetPhotoService` | presign 생성, 등록, 목록(viewUrl 포함), 삭제, 프로필 지정 |
| `io.bitpet.photo.controller.PetPhotoController` | `/api/v1/pets/{petId}/photos/**` |

### API 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| POST | `/api/v1/pets/{petId}/photos/presign?filename=` | presigned PUT URL 발급 |
| POST | `/api/v1/pets/{petId}/photos` | S3 업로드 후 DB 등록 |
| GET | `/api/v1/pets/{petId}/photos` | 목록 조회 (viewUrl 포함) |
| DELETE | `/api/v1/pets/{petId}/photos/{photoId}` | S3 + DB 삭제 |
| PATCH | `/api/v1/pets/{petId}/photos/{photoId}/profile` | 프로필 사진 지정 |

- `application.yml` — `bitpet.s3.*` 설정 블록 추가
